## 1. 项目背景与目标

### 1.1 背景

无缝钢管（Casing / Tubing）和筛管（Screen Pipe）是石油天然气井建设中的关键管材，均遵循 **API 5CT** 标准。当前缺乏一套集成的信息化管理系统来管理这两种管材的全生命周期——从采购、入库、库存管理到销售出库和质量追溯。

### 1.2 目标

构建一套基于 Rust 的 Web 管理系统，实现对无缝钢管与筛管的 **采购 + 销售 + 库存 + 质量** 一体化管理，覆盖管材从入厂到出厂的全流程数据追踪。

---

## 2. 目标用户与角色

| 角色 | 职责 | 核心关注点 |
|------|------|-----------|
| **仓库管理员** | 管材入库、出库、移库、盘点、标签打印 | 库存数量、库位信息、出入库操作效率 |
| **质量检验员** | 质量数据录入、质检报告管理、合格证关联 | 钢级/规格一致性、炉批号追溯、质检证书 |
| **销售/采购人员** | 采购订单管理、销售订单管理、客户/供应商管理 | 库存可售量、订单进度、合同信息 |
| **管理层** | 数据总览、报表查看、决策分析 | 库存周转率、库存金额、经营统计 |

系统为 **多用户系统**，支持基于角色的权限控制（RBAC）。

## 3. 功能需求

### 3.1 管材信息管理（P0 - Must Have）

**FR-PIPE-001：管材档案管理**
- **描述**：支持无缝钢管和筛管两种管材类型的增、删、改、查
- **无缝钢管属性**：
  - 管材编号（唯一标识）
  - 产品类型：套管（Casing）/ 油管（Tubing）
  - 钢级（Grade）：H40, J55, K55, N80, L80, C90, T95, P110, Q125 等
  - 规格尺寸：外径（OD）、壁厚（WT）、长度、单位长度重量
  - 端部类型：短圆螺纹（SC）、长圆螺纹（LC）、偏梯形螺纹（BC）、直连型等
  - 接箍参数：接箍类型、接箍外径、接箍长度
  - 炉批号（Heat Number / Lot Number）
  - 管体序列号
- **筛管特有属性**：
  - 筛管类型：绕丝筛管（Wire-wrapped）、割缝筛管（Slotted）、冲孔筛管（Punched）、金属棉筛管等
  - 基管参数：外径、壁厚、钢级（基管本身也是无缝钢管）
  - 缝宽/孔径规格
  - 过滤精度（如 150μm, 250μm 等）
  - 筛管长度与接头类型
- **通用属性**：生产厂家、生产日期、质检证书编号、附件（证书扫描件等）
- **验收标准**：
  - 支持新增/编辑/删除管材条目
  - 管材编号全局唯一
  - 支持按任意属性组合搜索
  - 筛管与无缝钢管分类型管理，但可在统一列表中查询

### 3.2 库存管理（P0 - Must Have）

**FR-INV-001：入库管理**
- 支持采购入库、生产入库、退货入库等多种入库类型
- 入库时记录：入库单号、入库日期、管材明细（编号/数量/批次）、供应商/来源、经手人
- 自动更新库存数量

**FR-INV-002：出库管理**
- 支持销售出库、领用出库、调拨出库等
- 出库时记录：出库单号、出库日期、客户/去向、管材明细、经手人
- 支持批次/单件出库（根据管材编号精确出库）

**FR-INV-003：库存查询与盘点**
- 实时库存查询：按管材类型、钢级、规格、库位等维度查看
- 库存变动明细（流水账）：每一件管材的出入库历史
- 库存盘点：生成盘点清单、录入盘点数据、生成盘点差异报告

**FR-INV-004：库位管理**
- 库区/货架/层位分级管理
- 管材与库位绑定，支持库位调拨

### 3.3 质量管理（P1 - Should Have）

**FR-QA-001：质检信息管理**
- 关联管材的质检报告/质保书
- 记录：检测项目、检测结果、检测日期、检测机构、检测人员
- 支持上传检测文件（PDF/图片）

**FR-QA-002：质量追溯**
- 通过炉批号（Heat Number）追溯到生产批次
- 通过管材编号追溯到单个管材的所有质量记录
- 关联 API 5CT 标准的钢级力学性能与化学成分参考数据，供对比校验

### 3.4 采购管理（P1 - Should Have）

**FR-PUR-001：采购管理**
- 采购订单创建、审核、跟踪
- 供应商信息管理（名称、联系人、资质等）
- 采购到货与入库联动

**FR-SALE-001：销售管理**
- 销售订单创建、审核、跟踪
- 客户信息管理
- 销售出库与库存扣减联动
- 查看库存可售量（Available-to-Promise）

**FR-CONTRACT-001：合同管理**
- 采购合同与销售合同的基本信息管理
- 合同与订单关联

### 3.5 数据导入导出（P1 - Should Have）

**FR-IO-001：数据导入**
- 支持 Excel / CSV 批量导入管材信息
- 导入时校验数据格式和必填字段
- 提供导入结果报告（成功/失败行数及原因）

**FR-IO-002：数据导出**
- 支持将查询结果导出为 Excel / CSV
- 支持导出库存报表、出入库明细、质量报告等标准报表

### 3.6 搜索与筛选（P0 - Must Have）

**FR-SEARCH-001：多维度搜索**
- 按管材编号、管材类型、钢级、规格（外径/壁厚）、状态、库位等条件组合查询
- 支持模糊搜索（管材编号片段、钢级名称等）
- 搜索结果分页展示

### 3.7 报表与统计（P2 - Could Have）

**FR-RPT-001：库存报表**
- 当前库存总览（按类型/钢级/规格汇总）
- 库存变动月报/季报

**FR-RPT-002：经营报表**
- 出入库统计图表
- 库存周转率分析
- 采购/销售趋势分析

### 3.8 标签打印（P2 - Could Have）

**FR-LABEL-001：条码/二维码标签打印**
- 根据管材信息生成条码或二维码标签
- 支持批量打印
- 标签模板可配置（显示字段、布局）

### 3.9 历史追溯（P0 - Must Have）

**FR-TRACE-001：全生命周期追溯**
- 记录每根管材从入库到出库的全部操作日志
- 记录每次修改的变更日志（谁、什么时候、改了哪些字段）
- 支持按管材编号查看完整生命周期

### 3.10 系统管理（P1 - Should Have）

**FR-SYS-001：用户与权限管理**
- 用户注册/管理
- 基于角色的权限控制（RBAC）
  - 仓库管理员、质量检验员、销售/采购人员、管理员
- 不同角色可见不同的菜单和操作按钮

**FR-SYS-002：操作日志**
- 记录用户的登录、数据修改等关键操作
- 日志可查询、可导出

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 |
|------|--------|
| 单页查询响应时间 | ≤ 2 秒（10 万条记录以内） |
| 数据导入性能 | 10 万条记录导入 ≤ 60 秒 |
| 并发用户数 | 支持 ≥ 20 人同时在线操作 |
| 系统可用性 | 99.5%（年度停机 ≤ 44 小时） |

### 4.2 数据规模

- 管材基础数据：数十万根（≥ 10 万条）
- 库存变动日志：百万级记录
- 数据存储方案：SQLite（需合理设计索引以支持大规模数据）

> **注意**：SQLite 适用于单机/小规模并发场景。对于多用户 Web 应用的写入并发，需关注 WAL 模式配置及连接池管理。如果未来并发需求增长，可考虑迁移至 PostgreSQL。

### 4.3 国际化与单位

- **界面语言**：支持中文和英文切换
- **单位系统**：支持公制（mm, kg/m, m）和英制（in, lb/ft, ft）切换
- 输入时可切换单位系统，内部存储统一为公制或包含单位标记

### 4.4 安全

- 用户密码加密存储（bcrypt / argon2）
- 敏感操作（删除/修改关键数据）需二次确认
- API 请求鉴权（JWT / Session）
- HTTPS（生产环境）

### 4.5 可维护性

- 基于 Rust 的类型安全系统，减少运行时错误
- 模块化架构，明确的包结构
- 后端提供 RESTful API，前端通过 HTTP 调用
- 提供 API 文档（OpenAPI / Swagger）

### 4.6 系统形态与技术栈建议

| 层级 | 建议技术 | 说明 |
|------|---------|------|
| **后端** | Rust + Axum + SQLx | Axum 是当前 Rust 生态最主流的异步 Web 框架；SQLx 编译期检查 SQL 语句 |
| **数据库** | SQLite（WAL 模式） | 零配置、文件级数据库，适合中小规模应用 |
| **前端** | React / Vue / Leptos | 前后端分离。如果追求全 Rust 技术栈可选 Leptos，否则 React/Vue 更成熟 |
| **API** | JSON REST API | 标准 RESTful 接口，便于与第三方系统集成 |

## 5. API 5CT 标准参考

> API 5CT（套管和油管规范）是石油天然气行业最重要的管材标准之一，以下内容为关键参考信息，用于指导系统的字段设计。

### 5.1 常用钢级（Grade）分类

| 钢级分组 | 钢级 | 类型 | 主要特征 |
|---------|------|------|---------|
| **H组** | H40 | 套管/油管 | 最低强度，非关键井 |
| **J/K组** | J55, K55 | 套管/油管 | 中等强度，中深井 |
| **N组** | N80 | 套管/油管 | 较高强度，N80-1 和 N80-Q 两种热处理状态 |
| **L组** | L80 | 套管/油管 | 抗腐蚀，含 Cr，用于 H₂S 环境 |
| **C组** | C90, C95 | 套管/油管 | 抗腐蚀，用于酸性环境 |
| **T组** | T95 | 套管 | 高抗挤毁，酸性环境适用 |
| **P组** | P110 | 套管/油管 | 高强度，深井 |
| **Q组** | Q125 | 套管 | 超高强度，超深井 |

### 5.2 端部类型

- **SC** (Short Round Thread) —— 短圆螺纹，用于较浅井
- **LC** (Long Round Thread) —— 长圆螺纹，连接强度高于 SC
- **BC** (Buttress Thread) —— 偏梯形螺纹，高连接强度
- **X** (Extreme Line) —— 直连型，用于特殊工况

### 5.3 计量单位

API 5CT 标准使用英制单位：

| 参数 | 英制单位 | 公制单位 |
|------|---------|---------|
| 外径 (OD) | inch (in) | 毫米 (mm) |
| 壁厚 (WT) | inch (in) | 毫米 (mm) |
| 长度 | foot (ft) | 米 (m) |
| 单位重量 | lb/ft | kg/m |
| 屈服强度 | psi | MPa |

### 5.4 标准规格参考

| 外径 (in) | 壁厚 (in) | 单位重量 (lb/ft) | 常用钢级 |
|-----------|----------|-----------------|---------|
| 4½ | 0.250 ~ 0.337 | 11.60 ~ 15.10 | J55, N80, L80 |
| 5½ | 0.304 ~ 0.415 | 17.00 ~ 23.00 | J55, N80, L80, P110 |
| 7 | 0.317 ~ 0.582 | 23.00 ~ 41.00 | J55, N80, L80, P110 |
| 9⅝ | 0.395 ~ 0.595 | 40.00 ~ 59.20 | J55, N80, L80, P110 |
| 13⅜ | 0.514 ~ 0.672 | 72.00 ~ 92.50 | H40, J55, K55 |

*（以上为部分常见规格示例，完整参考表应纳入系统参考数据）*

## 6. 数据模型初步设计

### 6.1 核心实体关系

```
                      ┌──────────────────────┐
                      │   SeamlessPipe        │  ← 无缝钢管（独立表）
                      │   (无缝钢管)           │
                      └────────┬─────────────┘
                               │
Supplier ──→ PurchaseOrder ──→ InboundRecord ──→ SeamlessPipe/ScreenPipe ──→ OutboundRecord ──→ Customer
                     ↑                                                          │
                Contract                                                   SalesOrder
                     │                                                          ↑
                     └──────────────────────────────────────────────────────────┘

                      ┌──────────────────────┐
                      │   ScreenPipe           │  ← 筛管（独立表）
                      │   (筛管)               │
                      └──────────────────────┘
```

> 无缝钢管（SeamlessPipe）和筛管（ScreenPipe）为**两个独立的数据表**，各有完整的字段体系，不共享表结构。

### 6.2 主要数据实体

| 实体 | 说明 | 核心字段 |
|------|------|---------|
| **SeamlessPipe** | 无缝钢管 | id, pipe_number, pipe_type(casing/tubing), grade, od, wt, length, weight, end_type, coupling_type, coupling_od, coupling_length, heat_number, serial_number, manufacturer, production_date, cert_number, location_id, status, created_at, updated_at |
| **ScreenPipe** | 筛管 | id, pipe_number, screen_type(wire-wrapped/slotted/punched), slot_size, filtration_grade, base_od, base_wt, base_grade, base_end_type, length, weight, heat_number, serial_number, manufacturer, production_date, cert_number, location_id, status, created_at, updated_at |
| **Location** | 库位 | id, zone_code, shelf_code, level_code, description |
| **Supplier** | 供应商 | id, name, contact, phone, address, qual_cert |
| **Customer** | 客户 | id, name, contact, phone, address |
| **PurchaseOrder** | 采购订单 | id, order_no, supplier_id, order_date, status, total_amount |
| **SalesOrder** | 销售订单 | id, order_no, customer_id, order_date, status, total_amount |
| **InboundRecord** | 入库记录 | id, record_no, type, pipe_type(seamless/screen), pipe_id, quantity, date, order_id, operator, remark |
| **OutboundRecord** | 出库记录 | id, record_no, type, pipe_type(seamless/screen), pipe_id, quantity, date, order_id, operator, remark |
| **InventoryLog** | 库存变动日志 | id, pipe_type(seamless/screen), pipe_id, change_type, quantity_before, quantity_after, operation, operator, timestamp |
| **QualityCert** | 质检证书 | id, pipe_type(seamless/screen), pipe_id, cert_no, inspect_date, inspector, agency, file_url, result, remark |
| **User** | 用户 | id, username, password_hash, role, name, email, language_pref, unit_system |
| **OperationLog** | 操作日志 | id, user_id, action, target_type, target_id, detail, ip_address, timestamp |
